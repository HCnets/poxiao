## 6. 第三方参考仓关系

### `_liquid_glass`

- 位置：`C:\Users\HCnets\Desktop\AI_REMOTE_TXCODEX\_liquid_glass\AndroidLiquidGlass-master`
- 结论：这是当前液态玻璃效果的直接源码参考仓。
- 原因：
  - 远程仓里包含 `backdrop` 库源码。
  - 当前本地 `app` 通过 Maven 依赖 `io.github.kyant0:backdrop:1.0.6`。
  - 本地 `app/src/main/java/com/poxiao/app/ui/LiquidGlass.kt` 已直接使用 `com.kyant.backdrop.*`。
- 建议用途：
  - 排查液态玻璃效果来源
  - 对照原库能力与本地二次封装差异
  - 出现视觉回归时作为底层参考

### `_miuix_main`

- 位置：`C:\Users\HCnets\Desktop\AI_REMOTE_TXCODEX\_miuix_main\miuix-main`
- 结论：这是旁路参考仓，不是当前 `Poxiao` 运行依赖。
- 原因：
  - 当前本地 `app` 未引入 `miuix` 或 `navigation3` 依赖。
  - 本地主工程源码搜索不到 `miuix` 或 `navigation3` 的实际使用。
- 建议用途：
  - 参考导航和组件形态
  - 评估后续是否值得迁移局部交互方案
  - 不建议在当前阶段直接纳入主工程基线

## 7. 推荐执行顺序

### 日常开发回归

1. 先跑本地已保留的 3 份文档
2. 若改动涉及待办、通知、导出、附件，再补跑对应远程文档
3. 若改动涉及玻璃特效，再对照 `_liquid_glass`
4. 若改动涉及导航/组件方案探索，再参考 `_miuix_main`

### 资料回收建议

1. 优先把远程独有的高价值测试文档补回本地
2. 不建议把 `_miuix_main` 整仓并入主工程
3. 保留 `_liquid_glass` 作为视觉效果底层参考仓
