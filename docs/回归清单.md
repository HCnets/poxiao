当前结论：

1. 当前可编译基线以 `app/src/main/java/com/poxiao/app/ui/PoxiaoApp.kt` 为准。
2. 后续如需再次拆分，必须采用更小步、可回滚、每步可编译验证的方式推进。

## 10. 下一轮拆分前必做检查

后续每次重新尝试页面拆分前，至少补跑以下链路。

### 10.1 App 壳层与页面切换

1. 启动应用，确认默认进入首页，无空白首屏、无闪退。
2. 依次点击底栏 `首页 / 课表 / 待办 / 番茄钟 / 更多`，确认页面切换正常、无内容错位、无点击失效。
3. 打开侧边导航后再关闭，确认不会卡住，也不会吞掉后续点击事件。
4. 在非首页状态按系统返回键，确认能回到首页；打开 overlay 时按返回键，确认优先关闭 overlay。
5. 从 `ReviewPlanner` 跳回首页历史聚焦后，确认首页能正常定位对应记录，不会跳错页面。

### 10.2 首页与首页快捷入口

1. 首页卡片、模块排序、首屏统计正常显示，无模块丢失。
2. 首页进入 `校园地图`、`校园服务`、`课程笔记`、`复习计划`、`智能体权限`，确认都能打开对应 overlay。
3. 首页进入 `今日课表`、`考试周`、`待办`、`番茄钟`，确认能跳到正确主页面并保留预期入口状态。
4. 首页触发复习桥接执行、历史回放、撤销后，确认文案、状态与列表刷新一致。

### 10.3 课表与考试周

1. 从首页跳入 `日视图`，确认课表页以 `Day` 模式进入。
2. 从首页跳入 `考试周`，确认课表页以 `ExamWeek` 工作台进入。
3. 在课表页打开 `课程笔记` 或 `教务账号` 相关入口，确认跳转链路正常。
4. 重启应用后再次进入课表页，确认缓存课表、附加事件和考试周完成状态可恢复。

### 10.4 待办、番茄钟与复习桥接

1. 智能体执行复习桥接后，确认待办中新增任务、番茄钟绑定标题、复习计划关联状态同时更新。
2. 回放历史执行记录后，确认新增待办、差异摘要和绑定任务状态正确。
3. 撤销最近一次复习桥接后，确认新增待办被移除，番茄钟绑定被清空，对应复习计划关联恢复。
4. 在待办页完成、顺延、编辑任务后，确认不会影响复习桥接历史读取。

### 10.5 通知与提醒调度

1. 修改通知偏好后，确认 `refreshLocalReminderSchedule()` 会重排提醒，不产生重复调度。
2. 待办提醒、课程提醒、考试周提醒同时存在时，确认最终调度列表能去重并限制在上限内。
3. 调整课表缓存或考试周完成状态后，重新触发调度，确认旧提醒会被取消，新提醒会重新建立。
4. Android 13+ 未授权通知权限时，确认本地通知发送会静默跳过，不闪退。

### 10.6 Overlay 页面返回链路

1. 从首页、更多页、侧边导航分别进入 `课程笔记`、`复习计划`、`导出中心`、`偏好设置`，确认返回后仍回到正确宿主页面。
2. 在 overlay 打开期间展开侧边导航或切主页面，确认不会出现页面叠层错乱。
3. 关闭 overlay 后底栏、岛屿提示、侧边导航按钮应恢复正常显示。

## 12. 下一轮安全拆分顺序

建议下一轮只做“小步拆分”，一次只拆一个低风险块，每步都要求 Android Studio 编译通过后再继续。

推荐顺序：

1. 先拆纯展示组件，例如无业务状态的卡片、按钮组、布局片段。
2. 再拆共享 UI 原子组件，但避免同时移动页面状态和副作用逻辑。
3. 再拆独立 helper 或纯函数，优先处理无 `Composable` 状态依赖的逻辑。
4. 最后才拆整页入口，例如 `HomeScreen`、`ScheduleScreen`、`TodoScreen` 这类高耦合页面。

执行约束：

1. 每次拆分只允许新增少量文件，不做整页批量外提。
2. 每次拆分完成后立即编译，必要时再补跑第 `10` 节回归链路。
3. 若出现跨文件大面积报错，优先检查字符串、括号、插值和可见性，不继续叠加修改。
