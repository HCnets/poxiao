# 本轮测试补充 - 智能体接入层

## 1. 默认星火链路不回归

### 测试步骤
1. 保持 `ASSISTANT_PROVIDER=SPARK`。
2. 打开首页智能体入口，发送一条普通文本消息。

### 预期结果
- 智能体仍然走原有星火链路
- 文本收发正常
- UI 与之前行为一致

## 2. Provider 配置切换

### 测试步骤
1. 将构建配置中的 `ASSISTANT_PROVIDER` 改成 `PREVIEW`。
2. 重新编译并发送一条消息。
3. 再将 `ASSISTANT_PROVIDER` 改成 `DEEPSEEK`。
4. 重新编译并发送一条消息。

### 预期结果
- `PREVIEW` 下返回本地预览文案
- `DEEPSEEK` 下不会崩溃
- `DEEPSEEK` 下会明确提示“接入位已预留”或“配置未完成”

## 3. DeepSeek 配置位

### 测试步骤
1. 检查以下构建字段是否可被环境变量或 `gradle.properties` 覆盖：
   - `ASSISTANT_PROVIDER`
   - `DEEPSEEK_BASE_URL`
   - `DEEPSEEK_API_KEY`
   - `DEEPSEEK_MODEL`

### 预期结果
- 四个字段都存在
- 不填时不会影响默认星火链路
- 后续只需补真实实现，不需要再改 UI 注入点

## 4. 语音与图片兜底

### 测试步骤
1. 在 `DEEPSEEK` provider 下测试语音和图片入口。

### 预期结果
- 不会闪退
- 会返回“语音入口尚未接入”或“图像入口尚未接入”的明确提示
